Creating NumPy Arrays from Scratch#

üîç Difference Between 1D and 2D Arrays in NumPy#

üü¶ 1D Array (One-Dimensional)#

A 1D array is like a simple list of values ‚Äî all arranged in a single row (or line).

üü® 2D Array (Two-Dimensional)#

A 2D array is like a table or matrix ‚Äî it has rows and columns.

üß† Key Differences Summary#

Feature

1D Array

2D Array

Structure

Single list

List of lists (rows & columns)

Shape

(n,)

(rows, cols)

Visualization

Horizontal line of values

Table/grid

Real-world Use

Vectors, signals

Matrices, images, spreadsheets

‚úÖ Notes#

  • np.full(shape, fill_value) lets you fill arrays with any number.

  • np.arange(start, stop, step) generates values like range().

  • np.linspace(start, stop, num) returns num evenly spaced points.

  • For 2D arrays using arange or linspace, use .reshape(rows, cols).

üé≤ The Old Way: np.random.seed() + RandomState#

np.random.seed(42)

np.random.rand(3)       # Uniform between 0 and 1
np.random.randn(3)      # Standard normal (mean 0, std 1)
np.random.randint(1, 10, size=3)  # Random integers
np.random.choice([1, 2, 3], size=2)
np.random.shuffle(arr)  # Shuffles inplace

üß™ The New Way: rng = np.random.default_rng(seed)#

rng = np.random.default_rng(42)

rng.random(3)           # Replaces np.random.rand()
rng.standard_normal(3)  # Replaces np.random.randn()
rng.integers(1, 10, size=3)  # Replaces np.random.randint()
rng.choice([1, 2, 3], size=2)
rng.shuffle(arr)        # Same name, but scoped to this RNG

üîÅ Side-by-Side Comparison#

Task

Old API (RandomState)

New API (Generator)

Functionality

Uniform [0, 1)

np.random.rand(3)

rng.random(3)

Draws samples from a uniform distribution over [0, 1).

Standard Normal

np.random.randn(3)

rng.standard_normal(3)

Draws samples from a standard normal distribution (mean = 0, std = 1).

Integers

np.random.randint(1, 10)

rng.integers(1, 10)

Returns random integers from low (inclusive) to high (exclusive).

Random Choice

np.random.choice(a)

rng.choice(a)

Selects a random element or sample from a given array.

Shuffle

np.random.shuffle(a)

rng.shuffle(a)

Shuffles a sequence in place (modifies the array directly).

Permutation

np.random.permutation(a)

rng.permutation(a)

Returns a randomly permuted sequence or array (does not modify original).

üí° Key Differences#

1. Names Change Slightly#

  • rand ‚Üí random

  • randn ‚Üí standard_normal

  • randint ‚Üí integers

2. Everything Is Scoped to Your RNG#

You can create multiple independent RNGs:

rng1 = np.random.default_rng(1)
rng2 = np.random.default_rng(2)

Each produces its own sequence of pseudo-random numbers ‚Äî no interference!

3. Better Algorithms Under the Hood#

  • Generator uses PCG64 by default ‚Äî it‚Äôs faster and statistically better than the old Mersenne Twister used by RandomState.

üß† Why This Matters#

Let‚Äôs say you‚Äôre simulating two models side-by-side. If you use the old np.random.seed(), random numbers for one model can interfere with the other.
But with default_rng(), each model gets its own private randomness, like having your own deck of shuffled cards üé¥.

üîç Example: Two RNGs Generating Different Sequences#
rng1 = np.random.default_rng(0)
rng2 = np.random.default_rng(0)

print(rng1.integers(0, 10, size=5))  # [7 4 7 7 3]
print(rng2.integers(0, 10, size=5))  # [7 4 7 7 3] ‚Äî same seed, same output

rng3 = np.random.default_rng(1)
print(rng3.integers(0, 10, size=5))  # Different output

‚úÖ Final Word#

Using default_rng() means:

  • Cleaner API

  • Better reproducibility

  • No global state

  • More reliable randomness for scientific computing

‚úÖ Using default_rng() Without a Seed#

import numpy as np

rng = np.random.default_rng()   # No seed given
x = rng.integers(1, 10, size=5)
print(x)

‚úî Works perfectly#

‚úî Produces high-quality random numbers#

‚úî Uses a random seed generated internally from system entropy#

Every time you run the code, you‚Äôll get different results (non-deterministic).

üß† How It Works#

When you call:

np.random.default_rng()

NumPy internally uses:

  • system entropy (like OS-level randomness)

  • to generate a fresh, unpredictable seed

  • for the PCG64 random number generator (modern, fast, statistically strong)

So the RNG is automatically seeded for you.

üéØ When Should You Avoid Using It Without a Seed?#

If you need:

  • Reproducible experiments

  • Same results each time

  • Debugging consistent behavior

Then you should provide a seed:

rng = np.random.default_rng(42)  # reproducible
