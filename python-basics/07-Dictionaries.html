Introduction to Dictionaries in Python#

üß† What Are Dictionaries in Python?#

A dictionary is an unordered collection of key-value pairs, where:

  • Keys must be unique and immutable (like strings, numbers, or tuples).

  • Values can be any type (mutable or immutable).

  • Syntax: { key: value }

‚úÖ Example#

student = {
    "name": "Alice",
    "age": 22,
    "major": "Physics"
}

‚öôÔ∏è Why Use Dictionaries?#

  • Fast lookups by key (O(1) average case).

  • Easy to model real-world mappings: name‚Üívalue, username‚Üíprofile, etc.

üß∞ Dictionary Methods in Python#

Method

Description

clear()

Removes all items from the dictionary

copy()

Returns a shallow copy of the dictionary

fromkeys(seq[, v])

Creates a new dictionary from keys in seq with value v

get(key[, default])

Returns the value for key, or default if key is not found

items()

Returns a view object of (key, value) pairs

keys()

Returns a view object of the keys

values()

Returns a view object of the values

pop(key[, default])

Removes specified key and returns its value

popitem()

Removes and returns the last inserted (key, value) pair

setdefault(key[, default])

Returns the value of key; inserts it if not present

update([other])

Updates the dictionary with key-value pairs from another dict or iterable

üß† Step-by-Step Breakdown#

1. Original dictionary creation#
student = {"name": "Alice", "age": 22}

Here, Python creates a dictionary object in memory (say, at address 0x1001) with two key‚Äìvalue pairs.

student ‚îÄ‚îÄ‚ñ∫ { "name": "Alice", "age": 22 }
2. Shallow copy creation#
new_student = student.copy()

The .copy() method creates a new dictionary object (say, at address 0x1002), but it copies references to the same values (not the same dictionary).

Now we have:

student ‚îÄ‚îÄ‚ñ∫ { "name": "Alice", "age": 22 }   (at 0x1001)
new_student ‚îÄ‚îÄ‚ñ∫ { "name": "Alice", "age": 22 }   (at 0x1002)

They look identical but are two different objects in memory:

id(student) != id(new_student)  # True
3. Modifying a top-level key#
new_student['name'] = "Binayak"

This change affects only the new dictionary, not the original one. Why? Because 'name' is a key in the top level of the copied dictionary, and changing it replaces the reference in new_student only.

Result:

student     ‚îÄ‚îÄ‚ñ∫ { "name": "Alice", "age": 22 }
new_student ‚îÄ‚îÄ‚ñ∫ { "name": "Binayak", "age": 22 }

Hence,

print(student)      # {'name': 'Alice', 'age': 22}
print(new_student)  # {'name': 'Binayak', 'age': 22}

üß© What if there were nested dictionaries?#

If your dictionary has nested mutable objects (like another dictionary or list), .copy() only copies the outer structure, not the inner ones.

Example:

student = {"name": "Alice", "details": {"age": 22, "city": "Delhi"}}
new_student = student.copy()

new_student["details"]["city"] = "Kolkata"

print(student["details"]["city"])  # Output: Kolkata üò≤

Here, both dictionaries share the same inner dictionary reference for "details". To avoid this, you need a deep copy.

üîç Deep Copy (for nested structures)#

Use the copy module:

import copy
new_student = copy.deepcopy(student)

Now both the outer and inner objects are independent in memory ‚Äî changes in one won‚Äôt affect the other.

üß† Summary#

Concept

Meaning

Shallow copy

Creates a new container but references the same inner objects

Deep copy

Recursively copies all objects inside

Top-level change

Affects only the copied object

Nested change

Affects both unless deep copied

Memory implication

id(student) != id(new_student) but their inner elements may share the same references

‚ö° Python Dictionary Performance Insights#

Dictionaries in Python are implemented using hash tables, which give them very fast access times ‚Äî on average O(1) for most operations.

‚öôÔ∏è Why Is Lookup So Fast?#

Python dictionaries are based on:

  • Hashing: The key is hashed to an integer (using __hash__()).

  • Hash Table: The hash is used to find the index where the value is stored.

  • Collision Resolution: If two keys hash to the same index, Python resolves it using open addressing (probing nearby slots).

üß† What Can Cause Worst-Case?#

  • Too many hash collisions.

  • A malicious user designing inputs with the same hash values.

  • Rehashing during dictionary resizing (when it grows beyond its capacity).

üß™ Real Tip: Use Dictionaries When‚Ä¶#

‚úÖ You need fast lookups, inserts, or deletes

‚úÖ You have uniquely identifiable keys (like IDs, usernames)

‚úÖ You want to map one value to another (e.g., country_code ‚Üí country_name)

‚úÖ You don‚Äôt care about the order ‚Äî though since Python 3.7, insertion order is preserved
