Introduction to File Handling in Python#

File handling is an important part of any programming language, as it allows you to store, retrieve, and manipulate data in files permanently (instead of losing it when the program ends).

In Python, file handling is done primarily using the built-in open() function.

Opening a File#

file_object = open("filename", "mode")

  • filename ‚Üí name or path of the file (e.g., ‚Äúdata.txt‚Äù or ‚ÄúC:/folder/data.txt‚Äù)

  • mode ‚Üí specifies what you want to do with the file (read/write/append, etc.)

Common Modes#

Mode

Meaning

Description

"r"

Read

Opens file for reading (default). Error if file doesn‚Äôt exist.

"w"

Write

Opens file for writing. Creates new file or overwrites existing.

"a"

Append

Opens file for appending. Adds data to the end.

"x"

Exclusive Creation

Creates a new file. Error if file already exists.

"t"

Text Mode

Default. Reads/writes text data.

"b"

Binary Mode

Reads/writes binary data (images, videos).

"r+"

Read + Write

Opens file for both reading and writing.

 
f = open('./Files/sample.txt', 'r')
print(f.read())
f.close()

Reading a File#

Python gives multiple methods to read file contents:

(a) read() ‚Üí Reads entire file#

print(f.read())   # prints everything
f.close()

(b) read(n) ‚Üí Reads n characters#

Closing a File#

  • Always close files after use to free resources.

  • But instead of manually closing, best practice is using with statement:

with open('./Files/sample.txt', 'r') as f:
    lines = f.readlines()
    for line in lines:
        print(line.strip())
# file auto closed here

üëç Let‚Äôs break down this line:

img = mpimg.imread(io.BytesIO(content), format='png')

üîπ Step by Step Explanation#

  1. content

    • This is the raw binary data of your image.

    • You got it from:

      with open("image.png", "rb") as f:
    content = f.read()

    • At this point, content is just bytes (b'....').

  2. io.BytesIO(content)

    • Wraps the raw bytes into a file-like object that behaves like a real file.

    • imread expects something that looks like a file (not just bytes).

    • So BytesIO acts like a virtual file in memory.

    ‚úÖ Example:

    import io
fake_file = io.BytesIO(content)

  3. mpimg.imread(...)

    • mpimg is matplotlib.image.

    • imread reads an image from a file (or file-like object) and returns it as a NumPy array.

    • This array contains the pixel values of your image.

      • For grayscale ‚Üí 2D array

      • For RGB ‚Üí 3D array (height √ó width √ó 3)

      • For RGBA ‚Üí 3D array (height √ó width √ó 4)

  4. format='png'

    • Explicitly tells imread the image format (in this case PNG).

    • It‚Äôs useful when reading from a BytesIO object because it doesn‚Äôt have a filename (so imread can‚Äôt guess the format from the extension like .png or .jpg).

    • Without it, you may get errors for certain formats.

‚úÖ So in plain words:#

This line takes the binary image data (content), turns it into a file-like object (BytesIO), and then matplotlib reads it as if it was a real .png file ‚Üí returning a NumPy array of pixels.

File Pointer and seek()#

The file object maintains a cursor (pointer).

  • tell() ‚Üí shows current cursor position

  • seek(pos) ‚Üí move cursor to position pos

f = open('./Files/sample.txt', 'r')
print(f.read(57))
print(f'The cursor position is at : {f.tell()}')
f.seek(0)              # reset cursor
print(f.read())        # read full again
f.close()

Hybrid File Modes in Python#

Mode

Meaning

Behavior

"r+"

Read + Write

Opens file for both reading and writing. File must exist. Cursor starts at beginning.

"w+"

Write + Read

Creates a new file (or overwrites existing). Allows reading too. Cursor starts at beginning.

"a+"

Append + Read

Opens file for appending and reading. File is created if it doesn‚Äôt exist. Cursor at end for writing, but you can still read.

 
with open("./Files/sample.txt", "r+") as f:
    content = f.read()
    print("Before:", content)
    f.write("\nNew line added!")   # writes at end (since cursor moved after read)
    f.seek(0)
    print("After:", f.read())
